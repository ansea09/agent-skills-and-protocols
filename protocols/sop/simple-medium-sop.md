# Simple-Medium SOP

This SOP explains the mandatory short checklist.

## Purpose

Use the simple-medium protocol for bounded low-risk tasks where a full multi-phase protocol would add overhead without materially improving safety or truthfulness.

## Step Explanations

1. Refresh FPF and protocol repository so the answer uses current governing material or explicitly disclosed cached material.
2. Classify the normalized task, because one user message can contain several requests and the surface length of a message does not determine complexity.
3. Identify systems, roles, method, work, and expected output to prevent category mistakes such as treating a document as the actor or a plan as completed work.
4. Check whether evidence is needed. If evidence becomes non-trivial, route to complex.
5. Answer in domain language so the user gets a useful answer rather than internal framework jargon.
6. Separate facts, assumptions, hypotheses, and recommendations so uncertainty is visible.
7. Verify consistency, time sensitivity, and factual grounding before finalizing.
8. Add an engineering basis when the answer is substantive so the user can audit what was used.
9. Mark completion so the checklist has a visible end.

## Output Shape

For small answers, do not print the full checklist unless the user asks. Execute it internally and include only the concise engineering basis.
